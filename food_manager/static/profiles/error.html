Error Response Profile

This profile defines the structure of error responses returned by the API. All error responses follow a consistent format and include a machine-readable and human-readable message.

Error Structure

{
  "@error": {
    "@message": "Short error title",
    "@messages": ["Detailed description of the error"]
  },
  "@controls": {
    "profile": {
      "href": "/static/profiles/error.html"
    }
  }
}

Fields

FieldTypeDescription
@error.@messagestringA short, human-readable title describing the error.
@error.@messagesarray of stringsAdditional detailed messages explaining the cause of the error.
@controls.profile.hrefstringPoints to this error profile for clients to understand the structure.

Common HTTP Status Codes

  • 400 Bad Request — Validation failed, missing fields, or schema mismatch.
  • 404 Not Found — Requested resource does not exist.
  • 409 Conflict — Duplicate entries or constraint violations.
  • 415 Unsupported Media Type — Content type must be application/json.
  • 500 Internal Server Error — Unexpected server-side failure.

{

  },
